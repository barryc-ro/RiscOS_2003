RCS Keywords

This is an list of some useful RCS Keywords. When a file including these keywords is checked into CVS, appropriate strings will be added in...
$Author$
The login name of the user who checked in the revision.
$Date$
The date and time (UTC) the revision was checked in.
$Header$
A standard header containing the full pathname of the RCS file, the revision number, the date (UTC), the author, and the state.
$Id$
Same as $Header$
$Name$
Tag name used to check out this file.
$Log$
The log message supplied during commit, preceded by a header containing the RCS filename, the revision number, the author, and the date (UTC). Existing log messages are not replaced. Instead, the new log message is inserted after $Log$. Each new line is prefixed with a comment leader which RCS guesses from the file name extension. It can be changed with cvs admin -c.
$RCSfile$
The name of the RCS file without a path.
$Revision$
The revision number assigned to the revision.
$Source$
The full pathname of the RCS file.
Last modified: $Date$
