Overview

srcnotify allows users to maintain their own list of which directories and branches produce notification when they change. For example a build master may be interested in all changes across all branches of the repository. An engineer might be interested whenever anybody else makes a change to a module they authored.

Usage

The following commands are provided :

Command Description
srcnotify List all current notifications.
srcnotify add Add a notification.
srcnotify remove Remove a notification.

srcnotify

All the current notifications associated with the current user will be listed.

srcnotify remove

All the current notifications will be listed with a identifier associated with each. The user will be prompted to enter a number to identify the notification to be removed.

srcnotify add

The user will be prompted to enter the branch name and directory on which notifications are to be applied. Sensible default responses (see below) are used if no input is given.

  • Directory (default : current directory) :

    Notify the user whenever commits are applied to any file in this directory and any directories below it.

  • Branch (default : trunk) :

    Restrict the notification to changes that are applied to any file in the specified directory only on this branch. Entering "all" will allow notifications across all branches.

The following command line flags can optionally be used. 

srcnotify [-q] [-h] [-r <branch>] -d [path] [add|remove]
Parameter Flag Description
Options -q Quiet operation, don't prompt for user input, take default options as those required.
Directory -d Use specified directory for notifications.
Branch -r Use specified branch for notifications.
Help -h Display usage help.
