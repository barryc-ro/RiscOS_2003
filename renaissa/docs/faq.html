Overview

This document is the Acorn cvs FAQ (Frequently Asked Questions). It is intended that it will be regularly updated as users questions are answered. This will become increasingly important as new staff start and/or people migrate from using !SrcFiler to cvs.

Please refer to this document for any support queries before hasseling the authors.

The Answers

UNIX cvs answers

  1. What is cvs ?

    cvs is a version control system. Using it you can record the history of you sources files.

  2. Where are cvs files held ?

    cvs maintains its files inside a centralised repository. This is a directory which is populated with a hierarchy of files and directories.

  3. Who can access cvs ?

    Anyone within the unix file permissions group 'renaissa' can check out and view files within the cvs repository. Checking in of modified files is more tightly controlled using an access list maintained by the cvs administrators. This access list allows a user to be granted permission to alter files on a particular cvs branch within a particular directory sub-structure.

  4. What is a branch ?

    A branch is a logical line of development typically used to isolate source files. A branch would be used for example to prevent unwanted modifications being applied to a set of source files during a freeze period. A branch will have an ancester - ie where the branch was taken from. Once finished with, a branch can be merged with another branch to allow changes to propagate.

  5. What is the trunk ?

    The trunk is a special case of a branch, it is also the default branch in some circumstances.

  6. I get the error 'cvs: command not found'

    The machine you are using either does not have cvs installed, or your path is not set correctly to locate the cvs application. Normally you will be using cvs on valhalla with /usr/local/bin on your path.

  7. I get the error 'cvs: Sorry, you don't have read/write access to the history file'

    You are not in the unix file permissions group 'renaissa'. Contact your cvs administrator and get them to sort it out.

RISC OS cvs answers

  1. What is RISC OS cvs ?

    RISC OS is a port of command line cvs. The port is only of the client-side part of the CVS program, so will only work with a remote repository on a server. This is sufficient for use with the main RISC OS build tree, which is served by valhalla.

  2. RISC OS requirements ?

    Internet module RISC OS cvs requires version (X.XX) or greater of the Internet module - there are some serious bugs in previous versions which cause crashes when used from cvs. These problems only seem to occur when running in a TaskWindow.

    WimpslotRISC OS cvs uses the UnixLib runtime system rather than that of the Shared C Library. This means that it cannot extend its wimpslot, and also seems to cause nasty crashes when it runs out of memory. Enough testing has not been done to ascertain the minimum amount of memory required, try a 2Mb wimpslot.

  3. How is Prefix used ?

    CVS often changes the CSD. If one has used '*prefix' however, then RISC OS will use the current value of prefix in preference to the CSD. Thus CVS will fail if prefix is set. In future, RISC OS CVS may use prefix for everything rather than the CSD, but currently this causes problems.

  4. How is the home directory specified ?

    CVS stores mildy-encrypted passwords inside a directory specified by the environmental value 'HOME'. To make this a little more RISC OS complient, RISC OS CVS uses <CVS_Home$Dir> in preference if it is set.

  5. How is the repository located ?

    To use CVS with the rennaisa repository on valhalla (which contains the complete RISC OS source tree) you should set <CVSROOT> to :

    :pserver:<username>&&valhalla:/projects2/renaissa/cvsroot
    where '<username>' is your username.
