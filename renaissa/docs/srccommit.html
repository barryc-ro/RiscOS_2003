srccommit instructions

Author(s): Rich Buckley

Overview

srccommit provides a wrapper to the cvs commit command. It has the ability to automatically maintain RISC OS module version numbers and provide uniqueness between modules along a branch. In addition, a single log entry will be associated with all the files changed within the module and sources are conveniently tagged with a unique identifier.

Usage

srccommit [-h] [-m message]
Parameter Flag Description
Message -m Specify log entry message on the command line rather than via an editor.
Help -h Display usage help.

When invoked srccommit will look for a file VersionNum in the current working directory.

VersionNum file not present

The following actions are performed if the VersionNum is not present.

  1. User is asked if a new file should be created, commit is aborted if it is not to be created.
  2. An initial version number is presented which can be accepted or modified by the user.
  3. A VersionNum file is created and added to the repository. If this is being performed on a branch the minor version number will be set to the cvs file revision number.

VersionNum file present

  1. The VersionNum file is read and the major module number is extracted.
  2. If working on the trunk, the user will be prompted for the major version number to be incremented. If on a branch, the minor version number will be automatically incremented.
  3. The user is prompted for a log entry to cover changes. The entry can be entered directly or via the configured editor (environment variable EDITOR). Alternativly the entry can be passed in via the -m flag.
  4. The sources are committed using the supplied log entry.
  5. The sources are tagged with a tag, constructed on the module name, major and minor version numbers.

File formats

The following is the format of the VersionNum file. It is constructed such that it can be included by both C and assembler module. 

; /* (0.01)
;    This file is automatically updated by srccommit, do not edit manually.
;    Can be included by both C and assembler.
;
			GBLS	Module_MajorVersion
			GBLS	Module_MinorVersion
			GBLS	Module_Date
Module_MajorVersion	SETS    "0.01"
Module_MinorVersion	SETS	"1.1"
Module_Date		SETS    "14 Aug 97"
		        END
 */
#define Module_MajorVersion     0.01
#define Module_MinorVersion	1.1
#define Module_Date      	14 Aug 97
;
; /* End */
It is the responsibility of the author to ensure that the constants defined in this file are used within their module as appropriate. It has been decided that the minor version number will be present following the module date string in the area currently used for free formatted text. This allows the origin of a particular source to be determined very easily.
