#
# Name:          Unicode Corporate Use Subarea as used by Adobe Systems
# Table version: 1.2
# Date:          22 Oct 1998
#
# Description:
#
#   This document lists the Unicode values (UVs) in the Corporate Use subarea
#   (CUS) that are used by Adobe Systems.  Refer to the document "Unicode and
#   Glyph Names," at
#   http://www.adobe.com/supportservice/devrelations/typeforum/unicodegn.html
#   for a description of the use of the Corporate Use subarea and glyph naming
#   conventions.
#
#   323 total Unicode values (UVs) are used. 40 are identical to Apple's
#   assignments, and 283 have been assigned by Adobe.
#
# Format: Semicolon-delimited fields:
#
#   (1) The Corporate Use subarea UV (4 uppercase hexadecimal digits).
#
#   (2) [optional] The Adobe Glyph List glyph name (upper- and lowercase
#       letters, digits). If this field is empty, then this character is not
#       in the Adobe Glyph List, and its glyph name should be of the form
#       "uni<CODE>" where <CODE> is the value of field 1. For example, the
#       glyph name for U+F66D, LATIN SMALL CAPITAL LETTER A WITH BREVE, should
#       be "uniF66D".
#
#   (3) A descriptive character name, using the same conventions as in the
#       Unicode Standard version 2.1 (uppercase letters, hyphen, space).
#
#   (4) [optional] The character decomposition, using the same conventions
#       as in the Unicode Standard version 2.1. Not all of them are full
#       decompositions, but recursive application of look-up for
#       decompositions will lead to a maximal decomposition. All
#       decompositions comprise standard Unicode characters; i.e. no
#       decompositions refer to characters in the Private Use subarea.
#
#       Several additional compatibility formatting tags are defined to
#       better describe some character decompositions:
#
#          <cap>        An accent or punctuation mark designed for use with
#                       capital letters.
#          <denom>      A character designed for use in the denominator of a
#                       vulgar fraction.
#          <numer>      A character designed for use in the numerator of a
#                       vulgar fraction.
#          <osf>        An old style figure or a character designed for use
#                       with old style figures.
#          <sc>         A small capital or a character designed for use
#                       with small capitals.
#
#   (5) [optional] Comments.
#
#   The entries are in increasing UV order.
#
#   Lines starting with "#" are comments; blank lines should be ignored.
#
# Revision History:
#
#   1.2  [22 Oct 1998]
#        138 characters to be used by some OpenType fonts were added.
#        The glyph "dotlessj" was also added.
#        The <capacc> compatibility formatting tag was renamed to <cap>.
#        A few descriptive names were slightly changed for consistency.
#        A few decompositions were changed to refer only to standard UVs.
#        The ordering of the section numbers in the comments was reversed.
#
#   1.1  [24 Nov 1997]
#        The descriptive name for glyph "onefitted" was changed from
#        "TABULAR DIGIT ONE" to "PROPORTIONAL DIGIT ONE".
#
#   1.0  [17 Jul 1997]  Original version
#
