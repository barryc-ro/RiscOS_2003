1.0 Overview

This document specifies the Reset module which handles user presses of the reset button on the rear panel of the NC100.

2.0 Outstanding issues

There are no outstanding issues.

3.0 Technical background

The NC100 has a reset button on the rear panel of the machine. The reset button performs the following functions:

  • When pressed while the NC100 is on the Reset module resets the NC.
  • When held in while the NC100 is powered on the OS Kernel resets all non-protected NV-RAM to factory settings.

Details of default NV-RAM settings and what locations are protected can be found in the NV-RAM Software Functional Specification[1].

When the Reset module initialises it claims the IOMD_Reset_DevNo device vector which is added to the Kernel for NC100. The interrupt routine which is attached to the vector simply calls SWI OS_Reset.

The Kernel is modified to detect if the reset button is held in when the NC100 is powered on.

8.0 External dependencies

There is a dependency on getting hardware to test the Reset function.

9.0 Acceptance test

  • The NC must reset when the reset button is pressed.
  • The NC must be reliable and not get confused with rapid pressing of the reset button.
  • Unprotected NV-RAM must be reset to factory defaults when the reset button is pressed and held in when power is applied to the NC.

10.0 Non compliances

This section is not applicable.

11.0 Development test strategy

The Reset module will be tested on an NC Model 1 by using the IOMD_Standby_DevNo to simulate the reset button using the front panel standby button.

12.0 Product organisation

The Reset module is written in C and forms part of the standard build tree. It uses the TinyStubs module to reduce static workspace allocation.
